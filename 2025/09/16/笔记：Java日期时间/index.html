笔记：Java 日期时间

Last updated on 2025-09-16T18:04:45+08:00

UTC 和时区

UTC (Coordinated Universal Time) 是世界协调时，是全世界统一的 “标准时间”，在本初子午线（格林尼治，经度 0°），记作 UTC+0

但是地球是圆的，不同地区的日出日落时间不同。如果全世界都直接使用 UTC，就会很混乱：比如英国人说 “我早上 7 点上班”，北京人也说 “我早上 7 点上班”，但实际上北京那时已经是下午，太阳快要落山了。

因此，人类引入了 时区（Time Zone） 的概念，用来规定各地的本地时间。世界被划分为 24 个时区，每个时区相对于 UTC 都有一个固定的偏移量，本地时间就是在 UTC 的基础上加上这个偏移量，通常写作 UTC+X 或 UTC-X

Unix 时间戳

最早的 Unix 时间戳指的是：从 1970-01-01 00:00:00 UTC 到某一时刻 经过的秒数，对应 UTC 时间线上的某个刻度。但是随着系统精度提高，很多语言/库（比如 Java、JavaScript）把时间戳扩展到了毫秒甚至是纳秒级

在 Java 中获取 Unix 时间戳的常用方式：

  1. 秒级 Unix 时间戳
    1. Instant 的 getEpochSecond
      1. 只截取秒数部分，截断了精度
    2. System.currentTimeMillis() / 1000
  2. 毫秒级 Unix 时间戳
    1. Instant 的 toEpochMilli
      1. 只截取毫秒级部分，截断了精度
    2. System.currentTimeMillis()
  3. 纳秒级 Unix 时间戳
    1. xxInstant.getEpochSecond() * 1_000_000_000L + xxInstant.getNano()，需要注意的是，不会丢失毫秒级，只是被包含在纳秒级时间戳里了

[!NOTE] 注意事项

  1. UTC 时间戳就是秒级 Unix 时间戳

java.time.Instant

Instant 概述

Instant 表示一个时间点，是一个 自 1970-01-01 00:00:00 UTC 起算的秒数 + 纳秒数，对应 UTC 时间线上的某个刻度

[!NOTE] 注意事项

  1. 它相当于 Java 里的纳秒级 Unix 时间戳
