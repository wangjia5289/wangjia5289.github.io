2. JUL

2.1. JUL 概述

JUL（Java Util Logging），是 JDK 内置的日志记录框架，使用时无需额外引入第三方库，使用相对简单。

JUL 虽然能满足一般应用的日志需求，但相比 Log4j、SLF4J 等第三方框架，其扩展性和高级特性较为有限。

2.2. JUL 核心组件

  1. Logger
    1. Logger（记录器）是日志框架的入口，我们可以通过调用 Logger.getLogger(String name) 获取 Logger 实例，并调用其方法记录日志信息。
    2. 需要注意的是：
      1. Logger 会根据我们设置的 Logger 日志级别来记录需要的日志，对于低于我们设定级别的日志，即使我们调用了日志的记录方法（如 logger.config(xxx)），这些日志也不会被 Logger 记录（可以理解为：日志的第一层过滤）
      2. Logger 通常搭配 Level 和 Filter 使用
  2. Filter
    1. Filter（过滤器） 用于在日志记录被发送到 Handler 之前进行筛选，可以进行定制，决定哪些日志信息应该被接受，哪些应被过滤掉（可以理解为：日志的第二层过滤）
  3. Handler
    1. Handler（处理器），负责接收 Logger 记录的日志，并将日志消息输出到指定目标
    2. 常用的 Handler 包括：
      1. ConsoleHandler
        1. 日志输出到控制台
      2. FileHandler
        1. 日志输出到文件
      3. SocketHandler
        1. 日志输出到网络端点
      4. MemoryHandler
        1. 日志暂存于内存，满足条件时批量输出
    3. 需要注意的是：
      1. 一个 Logger 可以关联多个 Handler，以便将日志消息输出到不同的目标。
      2. Handler 会根据我们设置的 Handler 日志级别来输出需要的日志，对于低于设定级别的日志，即使 Logger 将其传递过来，这些日志也不会被输出（可以理解为：日志的第三层过滤）
      3. Handler 通常搭配 Level 和 Formatter 使用
  4. Formatter
    1. Formatter（格式化组件） 定义了日志消息的输出格式。
    2. 常用的 Formatter 包括：
      1. SimpleFormatter
        1. 简单、易读的文本格式
      2. XMLFormatter
        1. 以 XML 格式输出日志
      3. 除此之外，我们可以通过扩展 Formatter 类来自定义输出格式
  5. Level
    1. Level（验证级别）定义了日志级别
    2. Level 的级别顺序为：
      1. OFF
        1. 禁止捕获所有级别的日志信息
      2. SEVERE
        1. 错误信息，表示严重错误或异常情况，应用可能无法继续运行
      3. WARNING
        1. 警告信息，提示潜在的问题，但应用仍然可以继续运行
      4. INFO
        1. 输出常规运行信息，提供系统运行状态
      5. CONFIG
        1. 输出配置项信息
      6. FINE
        1. 调试信息（少），详细调试时使用
      7. FINER
        1. 调试信息（中），详细调试时使用
      8. FINEST
        1. 调试信息（多），详细调试时使用
      9. ALL
        1. 捕获所有级别的日志信息

2.3. JUL 核心特性

JUL 是一个具备层次结构的日志记录器，采用父子关系进行组织。例如 com 是 com.example 的父级，而 com.example 又是 com.example.test 的父级，而最顶层的 com 则以根日志（ RootLogger） 作为其父级。

而 JUL 包含以下核心特性：

  1. 继承的特性
    1. 如果我们没有为子级 Logger 手动设置日志级别，它会默认继承父级 Logger 的日志级别
  2. 日志向上传递的特性
    1. 每当子级 Logger（例如 com.example）记录一条日志时，首先会通过自身配置的 Handler 输出
    2. 随后，这条日志会向上传递至父级 Logger，并通过父级的 Handler 输出（在父级同样会进行过滤）
    3. 这个向上传递过程会逐级进行，最终到达最顶层的 RootLogger，并通过其 Handler 输出
    4. 我们可以禁用这种向上传递行为（通过 setUseParentHandlers(false)）

  1. 由于日志是逐条向上传递的，而不是等到底层 Handler 整体输出后再整体向上传递，因此可能会看到类似以下的日志输出：

[!NOTE] 注意事项

  1. 如果需要将日志写入文件（即使用 FileHandler），需要注意以下三点：
    1. JUL 只创建文件而不会自动生成文件所在的目录，因此必须手动创建 logs 目录，并确保程序具有相应的读写权限。
    2. 文件路径的格式应使用 \\ 而非 \
    3. JUL 中的日志轮换机制流程如下：
      1. JUL 先将日志记录在 JUL.0.log 文件中。
      2. 当 JUL.0.log 文件写满时，它会被重命名为 JUL.1.log，同时创建一个新的 JUL.0.log 继续记录。
      3. 随着 JUL.0.log 日志再次写满，JUL.1.log 会被重命名为 JUL.2.log，而 JUL.0.log 则再次重命名为 JUL.1.log，这样依次进行。
      4. 当文件数量达到最大限制时，最早的日志文件（如 JUL.4.log）将被删除，以腾出空间

3. SLF4J

3.1. SLF4J 概述

SLF4J（Simple Logging Facade for Java）是一种日志门面框架，为各种日志框架提供统一的抽象接口（API），让开发者在应用中只需依赖于这一套 API，而无需针对不同的日志实现学习各自的专有 API

[!NOTE] 注意事项

  1. SLF4J 只是个日志门面，自身不提供任何配置功能
  2. 如果需要配置日志，例如设置日志级别等，仍需按照具体日志实现（如 Log4J2、JUL 等）的配置方式，进行相关配置

3.2. SLF4J 支持的日志级别

SLF4J 支持的日志级别顺序为：

  1. ERROR
    1. 输出错误信息，表示严重错误或异常情况，应用可能无法继续进行
  2. WARN
    1. 输出警告信息，提示潜在的问题（推荐）
  3. INFO
    1. 输出常规运行信息，提供系统运行状态
  4. DEBUF
    1. 输出调试信息，比如详细的配置解析过程
  5. TRACE
    1. 输出最详细的信息，几乎是逐行解释发生了什么

[!NOTE] 注意事项

  1. SLF4J 本身只是个日志门面，因此他没有默认的日志级别，其取决于我们使用的底层日志实现：
    1. JUL
      1. 默认日志级别是 INFO
    2. Log4J
      1. 默认日志级别是 ERROR
    3. Log4J2
      1. 默认日志级别是 ERROR

4. SLF4J + Log4J2

4.1. SLF4J + Log4J2 概述

使用 SLF4J 作为日志门面框架，搭配 Log4j2 的日志实现框架，绝对是当前市场上最强大、最灵活的日志功能实现方式。

4.2. Log4J2 核心组件

Log4j2 主要由 Loggers (日志记录器)、Appenders（输出控制器）和 Layout（日志格式化器）组成

  1. Loggers
    1. Loggers（日志记录器），负责记录日志，并控制日志的输出级别
    2. 需要注意的是，与 JUL 不同，Log4j2 中的日志信息在 Logger 中确定输出级别后，不会再经过 Handler 的再次过滤，这意味着 Logger 配置的输出级别直接决定最终的日志输出级别
  2. Appenders
    1. Appenders（输出控制器），负责把日志输出到指定位置
    2. 常用的 Appenders 包括：
      1. ConsoleAppender
        1. 将日志输出到控制台
      2. FileAppender
        1. 将日志写入到单一文件中
      3. RollingFileAppender
        1. 将日志写入到文件中，支持按大小和时间滚动
      4. JDBCAppender
        1. 将日志存储到关系型数据库中
  3. Layout
    1. Layout（日志格式化器），决定了日志的格式化方式，即日志输出的具体格式
    2. 常用的 Layout 包括：
      1. HTMLLayout
        1. 输出为 HTML 表格的格式
      2. JSONLayout
        1. 输出为 JSON 格式
      3. XMLLayout
        1. 输出为 XML 格式
      4. SimpleLayout
        1. 简单的日志输出格式
      5. PatternLayout（默认）
        1. 最强大的格式化组件，可以根据自定义日志输出格式

4.3. Log4J2 支持的日志级别

Log4J2 支持的级别顺序为：

  1. OFF
    1. 禁止捕获所有级别的日志信息
  2. FATAL
    1. 输出严重错误信息，应用可能无法继续运行
  3. ERROR
    1. 输出错误信息，表示严重错误或异常情况，但应用仍然可以继续运行
  4. WARN
    1. 输出警告信息，提示潜在的问题
  5. INFO
    1. 输出常规运行信息，提供系统运行状态
  6. DEBUG
    1. 输出调试信息，比如详细的配置解析过程
  7. TRACE
    1. 输出最详细的信息，几乎是逐行解释发生了什么
  8. ALL
    1. 捕获所有级别的日志信息
