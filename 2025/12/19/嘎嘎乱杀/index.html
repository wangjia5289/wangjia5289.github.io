1.2. 666

  1. 基本规范（名词、动词、副词、形容词）
    1. 名词
    2. 动词 + 名词
    3. be + 动词 + 名词
    4. 名词 + 动词 + 名词
    5. be + 名词 + 动词 + 名词
    6. 名词 + be + 动词 + 名词
    7. 名词 + be + 名词 + 动词 + 名词
    8. 需要注意的是：
      1. 动词：
        1. 动词 + 动词
        2. 为、是、像、有、将、给、让、导致（其他的都当副词，例如：用于、可以、但是、虽然、一定、要）
        3. 动词 + “的”
      2. 名词：
        1. 名词 + 名词（例如：我们通常可以在捕获并处理受检异常的过程中）
        2. 列出的文字中：在 AAAA 之前、在 AAAA 时、在 AAAA 之后、在 AAAA 中、在 AAAA 的过程中、等句子（需要注意的是：列出的文字中的 AAAA或整个列出的文字，要遵从规矩）（例如：我的意思是：如果我们不修改 Java 代码，虽然能实现 “保住 Java 程序”，但是不能实现 “保住任务”）
        3. waht 不需要遵循，但是 AAAA 必须遵循
        4. “AAAA” 整体算一个名词
      3. 形容词：
        1. 名词 + “的”、动词 + …（例如：最终让抛出该异常的任务执行成功）
      4. 如果名词前面有内容，那么必须为动词、形容词（例如：我们可以当捕获并处理低层异常，抛出一个高层的、贴近业务的异常）、Exception 具体类
  2. 省略
    1. 该 + 原名、… + 原名
  3. 其他替换
    1. 每一个 ➔ 每个
    2. 中 ➔ 中的
    3. 增 ➔ 记录、存储
  4. 因此、所以、因为、前面、但是，前面加句号
  5. 属性是记录，数据结构是存储（不清不楚的一律当数据结构处理）
  6. 通用的，哪些动词，放在那里都可以的，就不要放在动词一列，例如考虑，考虑 A 可以，考虑 B 也可以
