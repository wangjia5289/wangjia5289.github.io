3.1.1. Iterator＜T＞ iterator()

3.1.1.1. 方法概述

Iterator<T> iterator() 是一个接口方法，需要由我们的实现类进行具体实现，用于获取迭代器

这个方法看起来有些特别，我们以前常见的方法返回 void 或某个类的实例，但很少见返回一个接口类型（因为 Iterator<T> 是一个接口）。其实，这里返回的本质是一个实现了 Iterator<T> 接口的对象。

3.1.1.2. 方法实现

[!NOTE] 注意事项

  1. 这里要求我们返回一个实现了某接口的对象，而不是让当前类直接去实现该接口。这种设计很有意思，通常的做法是通过定义一个内部类来实现该接口（备忘录模式就是使用了这种方法）
  2. 当然，我们完全没必要从零造轮子，直接使用 Java 已经封装好的数据结构就行，比如 List、Set、Map
    1. 但其实我们只是写了个 “套壳产品”，内部完全依赖 List、Set、Map 做事，没有提供新的数据结构，也没有对 Collection 的行为增加额外的逻辑，外部却自称是一个新的 Collection 实现
    2. 这就是重复劳动，既没有复用现有实现，也没有增加实际价值，在学习阶段可以做一做

3.2. 接口方法（default）

3.2.1. default void forEach(Consumer＜? super T＞ action)

3.2.1.1. 方法概述

default void forEach(Consumer＜? super T＞ action) 是一个接口方法的默认实现，用于遍历当前 Iterable 实例中的所有元素（其实是迭代器中维护的元素集合），并对每个元素执行传入的操作逻辑

[!NOTE] 注意事项

  1. 增强 for 循环（for-each 循环）是 Java 提供的一种语法糖，在编译阶段会被编译器自动转换为等价的迭代器遍历代码
