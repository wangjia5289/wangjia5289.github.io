Last updated on 2025-08-27T09:56:56+08:00

1. 构造方法

1.1. 获取直接缓冲区 Buffer

直接缓冲区是指：Buffer 分配在操作系统的内存中，而不是 JVM 堆内存上。在 IO 操作中，直接缓冲区性能更高，因为它可以直接作用于本地系统的 IO。

从数据流的角度来看：

  1. 直接缓冲区的数据流
    1. 本地 IO ➔ 直接缓存 ➔ 本地 IO
  2. 非直接缓冲区的数据流
    1. 本地 IO ➔ 直接内存 ➔ 非直接内存 ➔ 直接内存 ➔ 本地 IO

需要注意的是，虽然 IO 性能更高，虽然直接缓冲区 IO 性能更高，但在分配和回收时开销较大。

[!NOTE] 注意事项

  1. 判断 Buffer 是否为直接缓冲区，可调用 xxBuffer.isDirect()，返回 true 就表示是直接缓冲区。

1.2. 获取非直接缓冲区 Buffer

非直接缓冲区是指：Buffer 分配在JVM 堆内存上
