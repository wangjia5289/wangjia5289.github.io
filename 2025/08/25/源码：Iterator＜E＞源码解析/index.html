1. Iterator ＜E＞ 概述

Iterator<E> 是一个接口，定义了 “一个集合如何支持按顺序取出元素” 的规范

2. 源码部分

3. 接口方法

3.1. 接口方法（无）

3.1.1. boolean hasNext()

3.1.1.1. 方法概述

boolean hasNext() 是一个接口方法，需要由我们的实现类进行具体实现，用于判断集合中是否还有下一个元素

3.1.1.2. 方法实现

3.1.2. E next()

3.1.2.1. 方法概述

E next() 是一个接口方法，需要由我们的实现类进行具体实现，用于返回集合中的下一个元素

3.1.2.2. 方法实现

3.2. 接口方法（default）

3.2.1. default void remove()

3.2.1.1. 方法概述

default void remove() 是一个接口方法的默认实现，用于删除迭代器返回的最后一个元素（上一次 next() 返回的元素）

[!NOTE] 注意事项

  1. 该方法的默认实现是直接抛出 UnsupportedOperationException 异常。
  2. 这意味着如果我们想要调用它，就必须重写该方法，而且在大多数情况下，我们可能并不需要实现这个操作。

3.2.2. default void forEachRemaining(Consumer＜? super E＞action)

3.2.2.1. 方法概述

default void forEachRemaining(Consumer＜? super E＞ action) 是一个接口方法的默认实现，用于将迭代器剩余的所有元素依次执行传入的操作
