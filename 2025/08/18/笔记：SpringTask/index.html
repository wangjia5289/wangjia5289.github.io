一、理论

Spring Task 概述

Spring Task 是 Spring 框架提供的任务调度和异步执行工具，它的核心作用是让你不用手动管理线程或定时器，也能轻松完成定时任务和异步任务的执行。

Spring Task 相关注解

@Scheduled

CRON 表达式

另外，CRON 表达式还可以包含一些特殊符号来定义更加灵活的定时规则

符号 含义 示例
* 每一个值 0 * * * * ? 表示 每分钟执行一次
, 枚举值 0 0,15,30,45 * * * ? 表示 每小时，第 0、15、30、45 分，第 0 秒 执行一次
- 区间 0 0 10-12 * * ? 表示 每天 10 点到 12 点，第 0 分，第 0 秒执行一次
/ 步长 0 0/5 * * * ? 表示 每小时，从 0 分开始，每 5 分钟执行一次
? 不指定
L 最后 0 59 23 L * ? 表示 每个月，最后一天第 23 个小时，第 59 分，第 0 秒，执行一次
W 日字段专用，表示离该日最近的工作日 0 0 10 15W * ? 表示 每个月，离15号最近的工作日，第 10 小时，第 0 分，第 0 秒，执行一次
# 星期字段专用，表示第几个星期几 0 0 9 ? * 2#1 表示 每个月，第 1 个星期的星期 1，第 9 小时，第 0 分，第 0 秒，执行一次

[!NOTE] 注意事项

  1. 在 CRON 表达式中，日和星期可能会产生冲突，因此至少其中一个需要使用 ? 表示 “不指定”

二、实操

快速上手

定时任务

    1. Spring Web

开启支持任务调度

标注 @Scheduled 注解

@Scheduled 一般标注在 Service 层。需要注意的是，@Scheduled 要生效，类必须是由 IOC 容器管理的 Bean

[!NOTE] 注意事项

  1. 定时任务默认使用的线程池是 ThreadPoolTaskScheduler

异步任务

添加相关依赖

  1. Web
    1. Spring Web

开启支持异步任务

标注 @Async 注解

@Async 一般标注在 Service 层。需要注意的是，@Async 要生效，类必须是由 IOC 容器管理的 Bean

[!NOTE] 注意事项

  1. 异步任务默认使用的线程池是 SimpleAsyncTaskExecutor
