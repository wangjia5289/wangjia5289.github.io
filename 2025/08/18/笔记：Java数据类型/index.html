[!NOTE] 注意事项

  1. 数据类型转换
    1. 赋值时
      1. 小数据类型的值赋值给大数据类型的变量时，Java 会自动进行类型转换
        1. byte → short → int → long → float → double
        2. char → int → long → float → double
        3. 由于 long 为 8 字节，float 为 4 字节，将 long → float 时，超出 float 精度范围的部分就会被舍弃，导致精度丢失
      2. 大数据类型的值赋值给小数据类型的变量时，必须进行强制类型转换
        1. 大的整数类型转换为小的整数类型时，会发生截断，只保留低位字节，数值可能发生较大变化
        2. 浮点数转换为整数时，小数部分会被直接截掉（不是四舍五入）
    2. 算术时
      1. 算术运算中，如果操作数类型不同，Java 会将它们自动提升到最高精度的类型再计算
      2. 需要注意的是，在算术前，byte、short、char 会先提升为 int。因此，只要涉及到这三种类型，结果至少是 int。例如：short + float → int + float → 结果为 float
  2. 建议在书写 long 和 float 的值时养成习惯加上后缀 L 和 F，这样能避免歧义，提高代码可读性
  3. 为 char 类型赋值时，例如 char c = 'A';，实际存储的是字符对应的 Unicode 编码。
    1. 输出时，如果类型是 char，Java 会自动将该编码转换回对应字符，因此输出仍然是 'A'
    2. 如果将 char 类型转换为 int 并输出，结果就是对应的 Unicode 编码值，输出的就是 65

四、引用数据类型

1. 类（class）

1.1. 具体类

1.1.1. 简单类（POJO 类）

1.1.1.1. POJO 类概述

POJO 类（Plain Old Java Object） 全称为 “普通的老式 Java 对象”，这个术语最早是为对抗 EJB（臃肿的企业 Java Bean）而提出的，旨在回归简单、纯粹的 Java 编程风格

其特点如下：

  1. 不实现特定的接口
  2. 不继承任何特定的父类
  3. 不包含业务逻辑或复杂方法
  4. 仅包含属性、构造方法（无参构造、有参构造）、Getter、Setter、equals、hashCode、toString 等基本方法

这种类通常用于数据封装与传递，可以简单理解为：“它啥也不干，只负责装数据、传递数据”。

1.1.1.2. Entity 类（DO、PO）

Entity 类通常用于映射数据库表，一个 Entity 对象对应表中的一条记录，常见于 ORM 框架，如 Hibernate、JPA、MyBatis。

1.1.1.3. DTO 类

DTO 类专门用来 在不同系统或不同层之间传递数据。例如，在微服务架构中用于跨系统调用，或者前端向后端发送请求时，后端接收并返回 DTO 对象给前端。

1.1.1.4. VO 类

VO 类专门用来 展示给前端或视图层的数据对象。

[!NOTE] 注意事项

  1. VO 更倾向于前端展示，DTO 更倾向于传输
1.1.1.5. BO 类

BO 类用于封装业务逻辑和业务数据，通常在业务层（Service 层）使用。如果业务层逻辑简单，可以直接用 Entity 代替 BO；当业务复杂时，BO 则发挥重要作用。

1.1.2. 复杂类

复杂类是我们平时编写的类，它们不像 POJO 类那样有严格限制，通常包含更多业务逻辑或功能。

1.1.3. 包装类

Java 是一门面向对象的语言，但它的基本数据类型（如 int、char 等）并不是对象，所以 Java 提供了包装类让这些基本类型也能在需要 “对象” 的场景中被使用：

  1. 包装类可以为 null
    1. 这在 ORM 框架中非常有用，因为你也不确定数据库里到底有没有这个值，会不会是 null
  2. 包装类可以存入集合
    1. 集合中只能存入对象，不能存入基本数据类型
    2. 而我们使用包装类，就可以直接存入集合中
  3. 包装类可以使用更多的方法

1.2.1. 类的继承

1.2.1.1. 匿名内部类继承

匿名内部类是四种内部类之一，主要用于继承某个类或实现某个接口。它的特点是不需要单独定义类名（因为是匿名的），因此能让代码更加简洁。

判断是否是匿名内部类，可以看 new 的写法：

  1. new Thread()
    1. 只是创建一个 普通的 Thread 对象
  2. new Thread() { … }
    1. 创建的是 匿名内部类对象

在编译时，编译器会自动生成两个字节码文件，一个是 Demo.class，另一个是 Demo$1.class。这个自动生成的类继承了 Thread 类，并重写了 run 方法。

当执行到 Thread myThread = new Thread("myThread") { ... }; 时，JVM 会像处理普通类一样，将 Demo$1.class 加载到方法区，并在堆区创建这个对象实例，赋值给变量 myThread。

[!NOTE] 注意事项

  1. 使用匿名类创建了临时的类对象，方法执行完后如果栈帧中没有对该对象的引用，它可能会在下一次 GC 时就被回收
  2. 但它对应的类信息仍然会加载到方法区，而且我们知道方法区中的类卸载条件非常严格，所以必须明确匿名类的适用范围：
    1. 如果只是临时封装逻辑，使用匿名类是合适的
      1. 但如果需要在两处及以上复用同一段逻辑，匿名类就不太合适
    2. 匿名类不支持灵活传参，若需要传参灵活，建议避免使用匿名类
    3. 在事件驱动或回调场景中，匿名类则非常适用
1.2.1.2. 显示声明类继承

显式声明类，是指无论你是直接定义一个类对象，还是在某个类中编写成员内部类、静态内部类、局部内部类等嵌套类，总之你都是明确地定义了一个具名类

[!NOTE] 注意事项

  1. 类只能继承一个父类，但是能实现多个接口
  2. 在 Java 中，所有类最终都继承自 java.lang.Object。即使在定义类时没有显式写出 extends。Object 类作为祖宗类，提供了一些常用的方法：
    1. toString()
    2. equals(Object)
    3. hashCode()
    4. getClass()
    5. clone()
    6. wait()/notify()/notifyAll()
    7. finalize()
    8. 这些方法，有一些你看不到它的代码块，这是因为他是 native 方法，是使用 C / C++ 等底层语言在 JVM 内部实现的
  3. 父类的属性和方法在访问修饰符选择上需要仔细斟酌：
    1. public
      1. 天下皆友
    2. protected
      1. 同包 + 子女特权
    3. package-private
      1. 同包是朋友，子类靠边站
    4. pivate
      1. 独守秘密
  4. 当我们去 new 一个子类对象的时候，它会先去调用父类的构造方法
    1. 父类有无参构造
      1. 如果子类构造器中没有显示调用 super()，Java 编译器会在子类构造方法中的第一行自动插入一个 super 调用
    2. 父类无无参构造
      1. 子类必须在其构造方法中使用 super(…) ，向父类构造方法传入参数来显示调用父类的带参构造器
  5. 字段方面的继承
    1. 对于静态变量和静态常量，看似我们的子类继承了，但是其实我们访问的还是父类中定义的，修改的还是父类中定义的，内存地址都不变的
    2. 对于实例变量和实例常量，我们的子类是真实继承了的
    3. 我们可以在子类中自己声明一些父类没有的字段
  6. 方法方面的继承
    1. 子类可以通过 super.method() 的方法去调用父类的方法。但是如果父类方法没有被子类重写，那么子类中直接调用 method() 也是一样的效果
    2. 子类也可以去根据需要去重写父类的方法，这样再去调用这个方法，就不是默认的调用父类的方法了，而是调用我们重写的方法，想要调用父类的方法还是需要调用 super.method()
    3. 如果父类是抽象类，那么具体子类必须实现父类中所有的抽象方法
      1. 如果子类本身还是抽象类，则可以不去实现这些方法，把责任继续 “往下传”
      2. 如果父类继承了其他抽象类或实现了接口，但还没重写/实现的方法，所有未重写/实现的方法，都需要在最终的具体子类中重写/实现

2. 数组（Array）

2.1. Array

数组是 Java 内置的数据结构，用于存储固定大小且类型相同的元素。

[!NOTE] 注意事项

  1. 数组本身没有方法，只有 length 属性用于获取长度。

2.2. Arrays

Arrays 是 Java 为了弥补数组功能的限制，专门提供用于操作数组的工具类。
