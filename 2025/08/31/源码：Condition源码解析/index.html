源码：Condition 源码解析

Last updated on 2025-09-02T11:59:14+08:00

1. 概述

Condition 是一个接口，定义了线程间等待与唤醒的协调机制，配合 Lock 使用，相当于 Object 的 wait/notify/notifyAll 的升级版。

我们先来想象以下 synchronized 锁，它配合 Object 的 wait/notify/notifyAll 使用。但是有一个关键限制，一个 synchronized 锁（即一个对象监视器）只有一个等待队列（WaitSet），如果你有多个条件需要等待，你就无法精确控制唤醒哪一类等待者，因为他们都在同一个队列中等待。

而一个实现了 Lock 接口的对象中，可以创建出多个实现了 Condition 接口的对象，每个实现了 Condition 接口的对象都拥有自己独立的等待队列，可以根据情况想唤醒哪个 Condition 队列中的线程，就唤醒哪个 Condition 队列中的线程

[!NOTE] 注意事项

  1. 实现了 Condition 接口的类，必须自己维护一个等待队列
    1. 在 JUC 中，队列一般分为两种
      1. 同步队列：
        1. 竞争锁失败的线程，在这里阻塞
        2. 例如 AQS 中的 Sync Queue，Monitor 中的 EntryList
      2. 条件队列
        1. 条件未满足的线程，在这里阻塞
        2. 例如 AQS 中的 Condition Queue，Monitor 中的 WaitSet
    2. 需要注意的是，这里所说的 Condition Queue，特指 AQS 内部类 ConditionObject 维护的那条等待队列，而并非所有实现了 Condition 接口的类都叫做 Condition Queue
    3. 只不过，由于 AQS 的 ConditionObject 实现极具代表性，所以在各种资料和讨论中，所谓的 Condition Queue 往往都是在讲它。
  2. Condition 为Object 的 wait/notify/notifyAll 的升级版，在 Object 中，这些方法必须在 synchronized 块里调用，因为调用线程必须持有该对象的锁。同理，Condition 的实现也必须依赖某个锁来保证调用的合法性，因此，Condition 的语义天然决定了它只能与独占锁配合使用，而不适用于共享锁
  3. 由于实现 Condition 接口相当繁琐，我们通常不会选择去实现它，而是去使用 Java 中的内置实现，即 AQS 中的 ConditionObject

2. 源码部分

3. 接口方法

3.1. 接口方法（无）

3.1.1. void await()

void await() 是一个接口方法，必须由具体的实现类进行实现，用于 线程进入条件队列，并释放当前持有的锁，直到被唤醒（Condition 的 signal/signalAll）或被中断（Thread 的 interrupted）后响应中断

3.1.2. void awaitUninterruptibly()

void awaitUninterruptibly() 是一个接口方法，必须由具体的实现类进行实现，和 await 类似，但不会响应中断

3.1.3. long awaitNanos(long nanosTimeout)

long awaitNanos(long nanosTimeout) 是一个接口方法，必须由具体的实现类进行实现，用于 等待 nanosTimeout 纳秒

3.1.4. boolean await(long time, TimeUnit unit)

boolean await(long time, TimeUnit unit) 是一个接口方法，必须由具体的实现类进行实现，用于 等待 time unit

3.1.5. boolean awaitUntil(Date deadline)

boolean awaitUntil(Date deadline) 是一个接口方法，必须由具体的实现类进行实现，用于 等待到指定时间点

3.1.6. void signal()

void signal() 是一个接口方法，必须由具体的实现类进行实现，用于 唤醒一个等待在该 Condition 上的线程，该线程从条件队列转移到同步队列（同步器资源在此时释放这种极限情况下，也必须先转移到同步队列）

需要注意的是，唤醒的具体线程不保证是最早排队的，因为该节点可能在 signal 之前就被取消了（超时/中断），就没资格进入同步队列了

3.1.7. void signalAll()

void signalAll() 是一个接口方法，必须由具体的实现类进行实现，用于 唤醒在该 Condition 上等待的所有线程
