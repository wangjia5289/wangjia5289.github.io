源码：Lock 源码解析

Last updated on 2025-08-31T22:44:53+08:00

1. 概述

Lock 是一个接口，定义了显式锁的基本操作规范

2. 源码部分

3. 接口方法

3.1. 接口方法（无）

3.1.1. void lock()

3.1.1.1. 方法概述

void lock() 是一个接口方法，必须由具体的实现类进行实现，用于 获取锁，如果锁已经被别的线程占用，会一直阻塞，直到锁释放，被中断（Thread 的 interrupted）后不会响应中断

3.1.2. void lockInterruptibly()

3.1.2.1. 方法概述

void lockInterruptibly() 是一个接口方法，必须由具体的实现类进行实现，与 lock 类似，但被中断（Thread 的 interrupted）后可以响应中断

3.1.3. boolean tryLock()

boolean tryLock() 是一个接口方法，必须由具体的实现类进行实现，用于 尝试立即获取锁

3.1.4. boolean tryLock(long time, TimeUnit unit)

boolean tryLock(long time, TimeUnit unit) 是一个接口方法，必须由具体的实现类进行实现，用于 等待指定时间尝试获取锁

等待期间内，如果被中断（Thread 的 interrupted），可以响应中断

3.1.5. void unlock()

void unlock() 是一个接口方法，必须由具体的实现类进行实现，用于 释放锁

需要注意的是，如果忘记释放锁，就会造成其他线程永远抢不到资源（死锁），一般都会写在 finall 代码块中避免意外

3.1.6. Condition newCondition()

Condition newCondition() 是一个接口方法，必须由具体的实现类进行实现，用于 创建一个 Condition 对象，实现更精细的线程等待/唤醒

而一个实现了 Lock 接口的对象中，可以创建出多个实现了 Condition 接口的对象，每个实现了 Condition 接口的对象都拥有自己独立的等待队列，可以根据情况想唤醒哪个 Condition 队列中的线程，就唤醒哪个 Condition 队列中的线程
