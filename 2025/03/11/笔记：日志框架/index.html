笔记：日志框架

0、导图：Map：日志框架

1、JUL 框架

1.1、JUL 概述

JUL 全程 Java Util Logging，是 JDK 内置的日志记录框架，使用时无需额外引入第三方库，使用相对简单。虽然能满足一般应用的日志需求，但相比 Log4j、SLF4J 等第三方框架，其扩展性和高级特性较为有限，更适合中小型项目或对日志需求较为简单的场景。

1.2、JUL 组件介绍

  1. Logger：
    1. Logger（记录器）是日志框架的入口，我们通过调用 Logger.getLogger(String name) 获取 Logger 实例，并调用其方法记录日志信息。
      1. name：name 作为 Logger 的唯一标识，通常使用 全类名 或 包名 进行命名，以便于实现层级化日志管理（Logger 具有继承关系，子 Logger 默认会继承父 Logger 的日志配置和处理方式）
    2. Logger 会根据设定的 Logger 级别来记录需要的日志。对于低于设定级别的日志，即使你调用了日志记录方法（如 logger.config(xxx)），这些日志也不会被记录（需要注意的是，不要将 Logger.xxxx 方法等同于 System.out，因为它只是记录日志，是否能够输出还取决于 Logger 和 Handler 的级别）
  2. Filter：
    1. Filter（过滤器） 用于在日志记录被发送到 Handler 之前进行筛选，可以根据定制决定哪些日志信息应该被接受，哪些应被过滤掉。
  3. Handler：
    1. Handler（处理器），负责接收 Logger 记录的日志，并将日志消息输出到指定目标，常见的 Handler 包括：
      1. ConsoleHandler：日志输出到控制台
      2. FileHandler：日志写入文件
      3. SocketHandler：日志发送至网络端点
      4. MemoryHandler：日志暂存于内存，满足条件时批量输出
    2. 每个 Logger 可以关联一个或多个 Handler，以便将日志信息分发到不同的输出目标
    3. Handler 会根据设定的 Handler 级别来输出需要的日志。对于低于设定级别的日志，即使 Logger 将其传递过来，这些日志也不会被输出。
  4. Formatter：
    1. Formatter（格式化组件） 定义了日志消息的输出格式。JUL 提供了两种默认的 Formatter：
      1. SimpleFormatter：简单、易读的文本格式
      2. XMLFormatter：以 XML 格式输出日志
    2. 你可以通过扩展 Formatter 类来自定义输出格式
  5. Level
    1. Level（验证级别）定义了日志的输出级别，常见的级别顺序为：
      1. OFF：关闭所有日志记录
      2. SEVERE：严重错误（系统级错误）
      3. WARNING：警告信息（潜在问题）
      4. INFO：常规信息
      5. CONFIG：配置项信息
      6. FINE：调试信息（少），详细调试时使用
      7. FINER：调试信息（中），详细调试时使用
      8. FINEST：调试信息（多），详细调试时使用
      9. ALL：输出所有级别的日志记录

[!NOTE] 注意事项

  1. Logger 和 Handler 都具有各自的日志级别，可以简单将其视为两层独立的过滤机制。

[!NOTE] 注意事项

  1. 为什么设置 RootLogger 的日志级别即相当于设定全局日志级别。原因在于 Logger 具有继承性：除非为特定包单独配置 Logger 级别或调用 logger.setUseParentHandlers(false) 禁用父 Logger 的默认行为，否则所有 Logger 都将默认继承其父 Logger 的级别，而这些父 Logger 最终都是 RootLogger
  2. 如果需要将日志写入文件，由于 JUL 只创建文件而不会自动生成目录，因此必须手动创建 logs 目录，并确保程序具有相应的读写权限。此外，需要注意文件路径的格式应使用 \\ 而非 \
