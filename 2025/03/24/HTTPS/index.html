HTTPS

1. HTTPS 概述

HTTPS（HyperText Transfer Protocol Secure，超文本传输安全协议）是 HTTP 的安全版本，它通过 SSL/TLS 加密传输数据，防止被窃听、篡改和劫持。

2. HTTPS 工作流程

  1. ==用户发起请求==
    1. 浏览器向服务器发送一个 HTTPS 请求，请求的 URL 以 https:// 开头，这表示浏览器希望建立一个安全的连接。
  2. ==服务器返回 HTTPS 证书==
    1. 服务器收到请求后，将自己的数字证书返回给浏览器。这个数字证书通常由受信任的 CA（证书颁发机构）签发，证书中包含以下内容：
    2. 公钥：用于加密数据，可以理解为一把锁，而锁的对应私钥只有服务器持有。
    3. 颁发者信息：证明证书的合法性，说明是哪个 CA 签发的。
    4. 有效期：规定了证书的有效时间段。
    5. 域名信息：证书绑定服务器的域名，确保与请求的服务器匹配。
  3. ==浏览器验证证书==
    1. 浏览器接收到证书后会进行以下验证：
      1. 检查证书是否由受信任的 CA 签发
      2. 检查证书是否在有效期内
      3. 检查证书中的域名是否与当前访问的域名一致
    2. 如果任一项验证失败，浏览器会向用户显示警告，提示该连接可能不安全。
  4. ==浏览器生成会话秘钥==
    1. 浏览器随机生成一个会话秘钥（对称密钥），并利用证书中的公钥进行加密。
    2. 然后将加密后的会话秘钥发送给服务器。由于只有服务器持有相应的私钥，而黑客没有，所以即便黑客截获了加密后的会话秘钥，也无法解密，只能由服务器解密
    3. 注意：非对称加密的加解密速度较慢，通常仅用于密钥交换，而不适合大数据量传输。
  5. ==服务器解密会话秘钥==
    1. 服务器收到加密后的会话秘钥后，使用自己的私钥进行解密，获得浏览器生成的会话秘钥。
  6. ==后续通信==
    1. 一旦浏览器和服务器都获得相同的会话秘钥，后续所有数据传输都将使用该密钥进行加密和解密。这样即使数据在传输过程中被截获，也无法被攻击者解密
    2. 注意：对称加密的加解密速度快，非常适合大量数据的传输，同时保证通信安全。

3. HTTPS 证书

3.1. HTTPS 证书 概述

HTTPS 证书就是 SSL/TLS 证书，最初，HTTPS 使用的是 SSL（Secure Sockets Layer，安全套接字层） 进行加密，但 SSL 3.0 已被淘汰，现在主要使用 TLS（Transport Layer Security，传输层安全协议）。

名称 状态 说明
SSL 1.0 未发布
SSL 2.0 废弃 ❌ 存在严重漏洞
SSL 3.0 废弃 ❌ 容易受 POODLE 攻击
TLS 1.0 废弃 ❌ 2020 年起主流浏览器不再支持
TLS 1.1 废弃 ❌ 安全性不足
TLS 1.2 推荐 ✅ 目前最广泛使用
TLS 1.3 最优 🏆 更快更安全

3.2. HTTPS 证书 分类

  1. ==DV 证书（Domain Validation）==：
    1. 仅验证域名所有权，适合个人博客、小型网站
    2. 例如：Let’s Encrypt（免费）、ZeroSSL
  2. ==OV 证书（Organization Validation）==：
    1. 需要验证企业身份，适用于企业网站、电商平台
  3. ==EV 证书（Extended Validation）==：
    1. 最高级别的验证，浏览器地址栏显示企业名称，适合金融、银行、政府网站

3.3. 获取 HTTPS 证书

3.3.1. 通过证书颁发机构（CA）购买证书

选择一个受信任的证书颁发机构（CA），如 DigiCert、Comodo、GlobalSign、Symantec 等，通常是获取 HTTPS 证书的最常见方式，适用于大多数企业和网站。

3.3.2. 使用免费证书服务（Let’s Encrypt）
3.3.3. 通过云服务商获取证书

如果你使用了云服务（如 GoDaddy、Bluehost、阿里云、腾讯云等），这些提供商通常会提供一键获取 HTTPS 证书的服务，方便快捷。

==2.生成私钥（Private Key）==

  1. -algorithm RSA：
    1. 指定了生成的私钥使用的加密算法，这里使用的是 RSA（Rivest-Shamir-Adleman）算法
    2. RSA 是一种常见的公钥加密算法，广泛用于 SSL/TLS 证书中
  2. -out private.key：
    1. 指定生成的私钥文件保存的路径和文件名
  3. -aes256：
    1. 表示使用 AES 256 位加密 来加密私钥文件。

==3.创建证书签名请求（CSR）==
虽然自签名证书不需要 CA 签发，但我们仍然需要生成一个证书签名请求（CSR）来定义证书的属性。

  1. -key private.key：
    1. 指定用于生成 CSR 的私钥文件。
  2. -out server.csr：
    1. 指定生成的 CSRF 文件保存的路径和文件名

      [!NOTE] 注意事项

      1. 输入命令后，你需要输入一些关于证书的详细信息
        • Country Name (2 letter code)：国家名称（如 US）
        • State or Province Name (full name)：省份或州名称
        • Locality Name (eg, city)：城市名称
        • Organization Name (eg, company)：组织名称（可以是公司名或其他名称）
        • Organizational Unit Name (eg, section)：部门名称（可以为空）
        • Common Name (e.g. server FQDN or YOUR name)：域名或主机名（如 www.example.com 或 localhost）
        • Email Address：电子邮件地址

==4.生成自签名证书==
使用上一步生成的私钥和 CSR 文件来创建自签名证书。自签名证书的有效期可以通过 -days 参数指定，通常设置为 365 天（即一年）

  1. -in server.csr：
    1. 指定用于生成自签名证书文件的证书签名请求文件
  2. -signkey private.key：
    1. 指定用于生成自签名证书文件的秘钥文件
  3. -out server.crt：
    1. 指定生成的签名证书文件保存的路径和文件名
  4. -days 365：
    1. 自签名证书的有效期，以天为单位

==5.检查自签名证书==

  1. -in server.crt：
    1. 用于检验的自签名证书文件
  2. -text：
    1. 以文本格式显示证书的详细信息。
    2. 包括证书的各种字段，如有效期、发行者、主题、序列号、公钥信息等。
  3. -noout：
    1. 表示只显示证书的详细信息，而不输出证书的内容
